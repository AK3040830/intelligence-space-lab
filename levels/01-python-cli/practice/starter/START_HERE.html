第 01 关：本地练习包（Python + 终端 + JSON）

本页指导你在自己电脑上打开终端、进入本练习包文件夹、运行 Python。请先看完上一级目录里的 demo/index.html（卡片教程）。

1. 这一包是干什么的？

让你用 Python 读一个 JSON 文件（demo/task_status.json），在终端里看到几行输出；再用 verify.py 自动检查文件是否齐全、JSON 是否符合要求。

2. 需要安装 Python

本关必须安装 Python 3（建议 3.10+）。若尚未安装，请按负责人提供的安装指引操作（Windows 安装时建议勾选 “Add Python to PATH”）。

3. 怎么确认 Python 是否安装？

打开终端（见下节），输入：

python --version

若显示版本号即可。若提示找不到命令，在 Windows 上可再试：

py --version

若只有 py 能用，请把下面所有 python 换成 py。

4. 怎么打开终端？

Windows：开始菜单搜索「PowerShell」或「终端」。

macOS：打开「终端」App。

Linux：打开任意终端。

5. 怎么进入「当前目录」？

以下命令里的路径要改成你电脑上本练习包的真实路径（与本文件同级的文件夹就是 starter）。

Linux / macOS 示例：

cd /你的路径/intelligence-space-lab/levels/01-python-cli/practice/starter

Windows PowerShell 示例：

cd D:\你的路径\intelligence-space-lab\levels\01-python-cli\practice\starter

确认：当前目录下应能列到 demo 文件夹和 verify.py。

常见错误：若在 01-python-cli 或仓库根目录运行命令，会提示找不到 demo/hello_task.py。请一定要 cd 到 practice/starter。

6. 运行示例脚本

在已进入 starter 的前提下执行：

python demo/hello_task.py

若 python 无效，试：

python3 demo/hello_task.py

你应看到类似：任务ID、当前状态、当前进度、提示信息 四行输出。

7. 运行验收脚本

仍在同一目录执行：

python verify.py

成功时应输出：

[OK] 第 01 关验收通过

8. 常见错误速查

  • 不在正确目录：先 cd 到 practice/starter。
  • Python 没安装或未进 PATH：重装或试用 py。
  • JSON 格式写错：逗号、英文引号、括号要配对；可对照原始 task_status.json。
  • 找不到文件：确认 demo/hello_task.py 与 demo/task_status.json 仍在 demo 文件夹内。
