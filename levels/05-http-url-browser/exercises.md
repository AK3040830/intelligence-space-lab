## 练习 1：启动本地 HTTP 服务

**目标：** 确认 `http.server` 在指定端口监听，且终端保持运行。

**操作步骤：**

1. `cd` 到 `starter` 目录。  
2. 执行：`python3 -m http.server 8005`（Windows 可试 `py -m http.server 8005`）。  
3. 看到类似 `Serving HTTP on ... port 8005` 的提示。

**成功标志：** 终端无报错；浏览器能打开 `http://127.0.0.1:8005/`。

**常见错误：**

- **端口被占用**：换一个端口（如 `8006`），浏览器地址也要改成新端口。  
- **目录不对**：必须在包含 `index.html` 的 `starter` 里启动，否则根路径内容不对。

## 练习 2：访问 HTML 页面

**目标：** 理解「路径」对应服务器上的文件名。

**操作步骤：**

1. 保持 `http.server` 运行。  
2. 浏览器打开 `http://127.0.0.1:8005/hello.html`。  
3. 修改 `hello.html` 中一句文字，保存后 **刷新** 页面。

**成功标志：** 页面内容与修改一致；地址栏仍是 `http://` 开头。

**常见错误：**

- 改完 **没保存** 就刷新。  
- 仍用 **本地文件路径** 打开旧页 → 本关请用 **`http://127.0.0.1:8005/hello.html`** 验证。

## 练习 3：访问 JSON 文件

**目标：** 知道 JSON 也可以作为 HTTP 响应体被浏览器直接请求。

**操作步骤：**

1. 访问 `http://127.0.0.1:8005/data.json`。  
2. 用编辑器修改 `message` 或 `items` 里一项，保存后刷新。

**成功标志：** 浏览器里看到的 JSON 文本已更新（部分浏览器会格式化显示）。

**常见错误：**

- JSON **语法** 写坏（缺逗号、引号不配）→ 可能变成空白或报错，用编辑器 JSON 校验或对照原样修复。

## 练习 4：访问不存在的路径并观察 404

**目标：** 把 **404** 与「资源不存在」对上号。

**操作步骤：**

1. 访问 `http://127.0.0.1:8005/not-exist.html`（本练习包故意未提供该文件）。  
2. 阅读 **`missing-page-note.txt`**，对照浏览器里状态或提示语。

**成功标志：** 能口头说明：404 表示服务器**没找到**你要的路径/文件（本关为静态文件服务场景）。

**常见错误：**

- 以为 404 是「Python 坏了」→ 多数是路径或文件名写错；本关故意用不存在的文件名练习。
