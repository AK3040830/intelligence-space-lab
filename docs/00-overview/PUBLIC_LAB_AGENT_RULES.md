# 公开训练仓安全边界（Agent 与协作者必读）

**`intelligence-space-lab` 是公开新人训练仓库**，只用于课程、Demo、练习包、教学说明和低风险示例。后续可通过 **GitHub Pages** 等对外提供访问，用于新人训练与课程学习。

**默认前提：本仓库内容会被公开访问。** 任何自动化 Agent 或人类协作者在编写、生成或提交内容时，都必须按本文约束行事。

---

## 1. 仓库定位

本仓库 **是**：

- 面向新人的 **闯关式训练材料**（卡片教程、小测、练习包、`checklist.md` 等）；  
- **低风险、非生产** 的示例与说明；  
- 与正式主项目 **解耦** 的教学与 Demo 载体。

本仓库 **不是**：

- **正式工程仓库**（不承担线上服务交付职责）；  
- **生产部署仓库**（不含真实上线流水线与生产制品）；  
- **模型权重仓库**（不存放真实或大体积私有模型文件）；  
- **服务器配置仓库**（不含真实主机、内网拓扑的可执行配置）；  
- **私有项目文档仓库**（不替代内部 Wiki、保密设计文档）。

---

## 2. 禁止加入的内容

以下类型 **禁止** 出现在本仓库的任何路径（含 `levels/`、`site/`、`docs/`、脚本注释、示例 JSON 等）：

| 类别 | 说明 |
|------|------|
| **凭据与密钥** | 真实或疑似可用的 API Key、Token、Secret、OAuth 刷新令牌、私钥、证书私钥材料等。 |
| **主机与访问** | 真实服务器 IP、域名与内网地址组合、SSH 私钥、`~/.ssh/config` 片段、可直连生产的连接信息。 |
| **数据库敏感信息** | 真实数据库账号密码、生产连接串、含真实数据的导出。 |
| **私有源码** | 从正式项目仓库 **整段复制** 的未脱敏业务逻辑、内部模块名与实现细节（若教学需要，应改写为最小虚构示例）。 |
| **模型与大数据** | 真实模型权重、大体积私有数据集、不可公开的评测集。 |
| **内部运维细节** | 生产环境拓扑、真实告警规则、内部工单系统链接、仅员工可见的部署手册全文。 |
| **个人隐私** | 学生或员工的身份证号、手机号、住址、未脱敏学籍/人事信息、真实简历全文等。 |
| **其他不宜公开资料** | 合同条款、商务报价、安全漏洞未修复前的可利用细节等；**凡不确定是否适合公开，一律视为禁止**，须人工确认后再处理。

教学用的 **虚构** 主机名（如 `example.com`）、**占位** API Key（如 `sk-test-xxxx`）、**明显无效** 的示例连接串，须在文案中标明为示例，且不得与任何真实环境雷同到可误用。

## 3. Agent 工作原则

1. **默认公开**：Agent 必须默认 **本仓库内容会被公开访问**；不得假设「只有内网能看」。  
2. **范围限定**：只能生成或修改 **教学内容、Demo、练习包、说明文档** 及与本训练目标一致的低风险脚本说明。  
3. **禁止搬运敏感代码**：不得从正式项目仓库 **复制** 可能含业务逻辑、密钥、内部命名的源码或配置。  
4. **禁止编造真实配置**：不得虚构看似真实且可能被误用的服务器地址、账号、Token 格式（示例须明显为占位）。  
5. **禁止个人隐私**：不得写入或推断并写入任何可识别个人的隐私数据。  
6. **不确定则停工**：对「是否含敏感信息」「是否来自生产」「是否可公开」存疑时，**必须停止生成或提交**，并请求 **人工确认**；不得在说明中写「应该没问题」之类搪塞。  
7. **与课程规范一致**：在遵守本文的前提下，课程结构、顺序与形态仍须符合 [`COURSE_DESIGN_STANDARD.md`](COURSE_DESIGN_STANDARD.md)。
